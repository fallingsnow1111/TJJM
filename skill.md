## 四、建模使用建议(面向后续统计建模)

- 主键建议:
	- 年份(year)
	- 地区(province, 对省级清单)
	- 指标(metric)

- 建议建模层级:
	- 宏观年度序列: 人口 + GDP + 能源 + 污染物(2016-2025/2024)
	- 省级面板: 年度 30 省碳排放清单(2001-2022, 年份完整)
	- 国家分部门: 中国分部门核算清单(1997-2022)

- 预处理提醒:
	- 年份列为宽表结构, 需 melt 到长表
	- 中英文字段并存, 建议建立字段映射字典
	- 单位需统一(如 万吨标准煤, Mt CO2)

## 六、踩坑

- 终端 Python 对中文路径匹配时仍可能出现不稳定, 直接按中文关键词过滤容易误判。
- PDF 解析库在当前运行环境不可直接导入, 无法稳定抽取 PDF 正文内容。

## 七、解决方法

- 分组统计采用“路径层级分段 + 最大分组识别”方式, 避免直接依赖中文关键词匹配。
- PDF 文件先纳入索引并记录用途与位置, 正文抽取留到后续独立步骤处理。
- 输出一份 JSON 结构化索引作为事实源, 再由人工可读索引进行归纳, 降低乱码与格式风险。
